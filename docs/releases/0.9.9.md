# Videorc 0.9.9 Beta 1

Videorc 0.9.9 is the **fix batch + pipeline honesty** release: the Publish
pipeline reports what each run produced, Library import works with zero
setup, the FX1–FX9 findings batch, and the OAuth secret's move to
build-time injection.

## Scope

- **Publish pipeline outcome visibility** — cards read the latest artifact at
  ANY status: Ready (content inline) / Waiting for consent (state-aware copy —
  points at step 0's switch when cloud is ready, at step 0's *reason* when
  cloud AI is unavailable) / Failed / Not run; an "Audio extracted" strip
  names the produced file with Reveal in Finder. A finished consent-off run
  no longer masquerades as "Not run".
- **Library import default** — a blank output directory resolves to
  `default_recordings_dir()` (created on demand, same isolation env as
  recording); a configured-but-missing directory still errors with
  actionable copy. Regression tests + `smoke:session-ops`.
- **FX1–FX9 findings batch** (docs in the 0.9.8 findings plan): mic Enable
  flips the chip without a needless backend restart; expired session shows
  session-expired copy (not signed-out); extraction + Check quality toast
  their results; account dropdown closes on Escape/nav; shortcuts respect
  modals and work from aux windows; local-time session titles; scene chips
  spacing; source-fallback toasts proven and readable.
- **OAuth hardening (pre-public-repo)** — the YouTube Desktop-client secret
  left source; compiled in via `VIDEORC_BUNDLED_YOUTUBE_CLIENT_SECRET`;
  `release:validate:macos` fails closed when the built backend lacks the exact
  build-time secret — **exercised for the first time this release: PASS**.
- Dev-only: the single-instance quit now says why (installed app holds the
  lock).
